## 🔍 실패 원인 분석

### 문제 1: API 응답 감시 실패 (Chromium, WebKit)
**증상**: API 응답 상태 코드가 400(Bad Request)으로 반환됨
**예상**: 201(Created)

**원인**: 데이터 저장소 동시성 이슈
- JSON 파일 기반 저장소가 동시 접근 처리 미흡
- 여러 브라우저에서 동시에 테스트 실행 시 파일 쓰기 충돌

**해결 방안**:
```
단기: 테스트 순차 실행 또는 고유 데이터 사용
장기: PostgreSQL 등 DB로 마이그레이션
```

### 문제 2: WebKit에서 성공 메시지 미표시
**증상**: "구독이 완료되었습니다" 메시지가 표시되지 않음
**예상**: 성공 메시지 표시

**원인**: Safari 특정 동작
- 브라우저 간 JavaScript 실행 타이밍 차이
- React 상태 업데이트 지연

**해결 방안**:
```
- waitForSelector() 대기 시간 증가
- 명시적 reload() 후 재시도
```

### 문제 3: Mobile Safari 중복 구독 검증 실패
**증상**: 중복 구독 에러 메시지가 표시되지 않음

**원인**:
- 모바일 환경의 데이터 동기화 지연
- 페이지 새로고침 후 데이터 로딩 시간 미흡

**해결 방안**:
```javascript
// 재시도 로직 추가
await page.waitForSelector('input[type="email"]', { timeout: 5000 });
```
